# VoiceGuide Live (v0.2 — 확장 중심 재구축)

**지금 보고 있는 웹페이지 옆(사이드패널)에서, 화면을 실시간으로 읽고 음성으로
한 단계씩 안내하는** Chrome 확장. 기존 웹앱(v0.1)의 근본 한계 — 별도 탭,
수동 컨텍스트 전달, 낡은 스냅샷 — 를 해결한 새 아키텍처입니다.

## v0.1과 무엇이 다른가

| | v0.1 (apps/web + apps/extension) | v0.2 (이 확장) |
|---|---|---|
| 가이드 위치 | 별도 탭 (localhost:5173) | 사이드패널 — 페이지 **옆에** 상주 |
| 화면 컨텍스트 | 수동 4단계 전달, 1회 스냅샷 | **매 질문마다 자동 라이브 스냅샷** |
| 요소 지칭 | 텍스트 라벨만 | ref + 위치 + 상태, **화면 하이라이트** 📍 |
| 단계 완료 확인 | 무검증 stepIndex++ | 직전 스냅샷 대비 **변화 diff로 검증** |
| LLM | Mock 기본 | Claude API 직접 연결 (Sonnet/Haiku/Opus) |
| DOM 한계 보완 | 없음 | 스크린샷 → Claude vision 폴백 📷 |

## 설치

```bash
# 저장소 루트에서
npm install
npm run build --workspace=@voiceguide/guide-extension
```

(이 폴더의 `dist/`가 이미 빌드되어 있다면 바로 3단계부터)

1. Chrome → `chrome://extensions` → 개발자 모드 ON
2. "압축해제된 확장 프로그램 로드" → `apps/guide-extension` 폴더 선택
3. 아무 웹사이트에서 확장 아이콘 클릭 → 사이드패널 열림
4. ⚙️ 설정에서 **Anthropic API 키** 입력
   ([console.anthropic.com](https://console.anthropic.com)에서 발급.
   Claude Pro 구독과는 별개로 API 크레딧이 필요합니다)
5. 🎙️ 버튼을 누르고 말하거나, 텍스트로 질문

## 사용 흐름

```
질문 → 활성 탭 라이브 스냅샷(요소·위치·상태) → 민감정보 마스킹
     → Claude → 한 단계 안내(JSON) → TTS 음성 + 📍 해당 요소 하이라이트
"완료했어" → 새 스냅샷 + 변화 diff → 실제 성공 여부 검증 후 다음 단계
```

- **📷 버튼**: 다음 질문에 화면 스크린샷 포함 (canvas 앱 등 DOM으로
  부족할 때. 모델이 필요하면 먼저 요청하기도 함). **스크린샷은 마스킹되지
  않으니** 환자 정보 등 민감한 화면에서는 사용 금지.
- **퀵 버튼**: 완료했어 / 못 찾겠어 / 더 쉽게 / 🔊 다시 듣기 / 🔇 중단

## 프라이버시

- 입력 필드의 **값은 어떤 경우에도 수집하지 않음** (라벨/placeholder만)
- 모든 텍스트는 전송 직전 `PrivacyRedactor`(core 재사용)로 마스킹
  (이메일·전화·주민번호·카드·API키·비밀번호), 마스킹 건수는 패널에 표시
- API 키는 `chrome.storage.local`에만 보관, 대화는 저장하지 않음
- 스냅샷·스크린샷은 사용자가 질문한 순간에만 1회 수집 (상시 감시 없음)

## 알려진 한계 (정직하게)

- **100%는 불가능**: canvas 렌더링 앱(Google Docs 편집기 등),
  cross-origin iframe 내부, `chrome://` 페이지는 DOM으로 읽을 수 없음
  → 스크린샷(📷) 폴백 사용
- Web Speech STT의 한국어 인식 품질은 조용한 환경 기준. 시끄러우면 텍스트 입력
- TTS는 OS 내장 음성 품질에 의존 (Windows 한국어 음성 설치 권장)
- 응답 스트리밍 미구현 → 첫 음성까지 2~4초 (다음 최적화 대상)
- 매 턴 스냅샷 전송 = 토큰 비용 발생. 비용 절감은 Haiku 모델 선택
